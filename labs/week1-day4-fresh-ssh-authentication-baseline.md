# Week 1 Day 4 - Fresh SSH Authentication Baseline

## Objective

This lab documents a fresh SSH authentication baseline review in a controlled home SOC lab environment.

The goal was to generate new SSH authentication events and review failed logins, successful logins, invalid username activity, source IP information, and a short authentication timeline.

## Lab Environment

- Hostname: `soclab`
- Target System: Ubuntu Linux VM
- VM IP Address: `192.168.64.7`
- Source IP: `192.168.64.1`
- Log Source: Linux journal logs
- Service: SSH

## Event Generation

A controlled failed-login test was generated by attempting to authenticate with the invalid username `fakeuser`.

A successful SSH login was then generated by authenticating with the valid user `analyst`.

## Commands Used

```bash
journalctl --since "15 minutes ago" | grep -i "Failed password" | wc -l
journalctl --since "15 minutes ago" | grep -i "Accepted password" | wc -l
journalctl --since "15 minutes ago" | grep -i "Invalid user" | wc -l
journalctl --since "15 minutes ago" | grep -Ei "Failed password|Accepted password|Invalid user" | grep -oE 'from [0-9]+\.[0-9]+\.[0-9]+\.[0-9]+' | sort | uniq -c
journalctl --since "15 minutes ago" | grep -Ei "Failed password|Accepted password|Invalid user" | tail -n 30
```

## Observed Results

- Failed password entries: `3`
- Accepted password entries: `1`
- Log lines containing `Invalid user`: `5`
- Source IP observed: `192.168.64.1`

## Timeline Sample

```text
May 10 14:59:22 soclab sshd[23736]: Invalid user fakeuser from 192.168.64.1 port 57213
May 10 14:59:35 soclab sshd[23736]: Failed password for invalid user fakeuser from 192.168.64.1 port 57213 ssh2
May 10 15:00:04 soclab sshd[23736]: Failed password for invalid user fakeuser from 192.168.64.1 port 57213 ssh2
May 10 15:00:08 soclab sshd[23736]: Failed password for invalid user fakeuser from 192.168.64.1 port 57213 ssh2
May 10 15:00:08 soclab sshd[23736]: Connection closed by invalid user fakeuser 192.168.64.1 port 57213 [preauth]
May 10 15:00:29 soclab sshd[23739]: Accepted password for analyst from 192.168.64.1 port 57215 ssh2
```

## Analysis

The logs showed `3` failed SSH password entries targeting the invalid username `fakeuser`.

The activity came from source IP `192.168.64.1`, which was the host machine in the lab environment.

A successful SSH login for the valid user `analyst` occurred shortly after the failed attempts.

In this controlled lab, the successful login was expected. In a real SOC environment, a successful login after repeated failed attempts should be reviewed to determine whether it was expected and authorized.

## Important Observation

The `Invalid user` count does not necessarily represent separate failed login attempts.

In this lab, the command counted log lines containing the phrase `Invalid user`. This included different types of SSH log messages, such as:

- Initial invalid user detection
- Failed password entries for an invalid user
- Connection closed messages involving an invalid user

For this reason, `Failed password` entries and `Invalid user` log lines should be interpreted separately.

## Key Takeaways

- `Failed password` entries indicate failed SSH authentication attempts.
- `Accepted password` entries indicate successful SSH authentication.
- `Invalid user` log lines may appear in multiple SSH message types.
- Source IP extraction helps identify where authentication activity came from.
- A successful login after failed attempts should be reviewed carefully in a real SOC environment.
